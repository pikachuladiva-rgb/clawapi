Authentication

All requests need Authorization: Bearer 

Authorization: Bearer <your-api-key>
GET
List Projects
curl https://HOST/v1/projects -H "Authorization: Bearer KEY"
GET
Get Project
curl https://HOST/v1/projects/PID -H "Authorization: Bearer KEY"
POST
Create Project
curl -X POST https://HOST/v1/projects \
  -H "Authorization: Bearer KEY" \
  -H "Content-Type: application/json" \
  -d '{"name":"My Bot","model":"claude-sonnet-4-6","system_prompt":"You are helpful."}'
DELETE
Delete Project
curl -X DELETE https://HOST/v1/projects/PID -H "Authorization: Bearer KEY"
POST
Create Session

Required before sending messages. Returns a session ID.

curl -X POST https://HOST/v1/projects/PID/sessions \
  -H "Authorization: Bearer KEY" \
  -H "Content-Type: application/json"
GET
List Sessions
curl https://HOST/v1/projects/PID/sessions -H "Authorization: Bearer KEY"
DELETE
Delete Session
curl -X DELETE https://HOST/v1/projects/PID/sessions/SID -H "Authorization: Bearer KEY"
GET
List Messages
curl https://HOST/v1/projects/PID/sessions/SID/messages -H "Authorization: Bearer KEY"
POST
Send Message

Uses Agent SDK by default — enables web search, web fetch, and other tools.

curl -X POST https://HOST/v1/projects/PID/sessions/SID/messages \
  -H "Authorization: Bearer KEY" \
  -H "Content-Type: application/json" \
  -d '{"content":"Hello!"}'

To use legacy Messages API instead (no tools): add "agent_sdk": false
